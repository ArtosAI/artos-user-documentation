Once document generation is complete, it becomes available for review and edits in your workspace. The document viewer sidebar has two tabs:

| Tab | What it's for |
| --- | --- |
| Details | Document Review: View information about the document (template used, source files, generation details) |
| Edit | Document Editing: Make changes to selected content |

## Document Review

### Section Selector

At the top of the **Details** tab, a dropdown shows which section you're viewing. Selecting text in the document will automatically update the Details tab to show the decision trail for that section.

### Decision Trail

Highlighting any text while reviewing a generated document surfaces the template-configured actions that contributed to drafting that passage. Highlight a minimum of 5 words.

### Action Cards

Each action card shows:

- **Indicators** for whether Expandable and Removable toggles are on
- **Action Details panel:** User instructions configured for the action
- **Decision Notes panel:** The rationale behind each decision:
  - **Completed** — Instructions fully followed
  - **Incomplete** — Instructions partially followed, with explanation
  - **Missing** — Instructions could not be followed due to missing source data
- **Source Selections panel:** Excerpts from source documents referenced when drafting the passage. Click an excerpt to open its source document.
- **Discrepancies panel:** Instances where configured instructions conflict with source document content
- **Style Guide panel:** Details on formatting instructions applied during generation

### Default Action Card Behaviour

| Action Type | Panel Expanded by Default |
| --- | --- |
| Use Template Text | Action Details |
| Fill in the Blank | Source Selections |
| Copy from Source | Source Selections |
| Copy & Modify from Source | Source Selections |
| Summarize from Source | Source Selections |
| Use Template Table | Decision Notes |
| Others | Source Selections |

### Best Practices

- Use **Source Selections** to verify the right excerpts were referenced
- Check **Decision Notes** when a passage doesn't read as expected
- Review **Discrepancies** to identify where your instructions and source content conflict — use them as a signal to tighten instruction wording
- Use **Style Guide** to confirm formatting instructions were applied correctly

## Document Editing

Make corrections and refine output in the **Edit** tab. Updates to one section will not affect the rest of the document.

The Edit tab has two sub-tabs:

| Sub-tab | What it does |
| --- | --- |
| Content | Edit the text content (length, tense, custom changes) |
| Source Document | Change which source document is used for the selected section |

### Select and Regenerate

Select any text in the **Content** sub-tab and ask Artos to rewrite it. Provide specific instructions under **Custom Instructions**, or choose from **Quick Actions** for length and tense changes.

<Note>
  If selecting a table, highlight the whole table. If selecting text, highlight more than 5 words.
</Note>

**Length:**

| Button | What it does |
| --- | --- |
| Longer | Regenerates with more detail and explanation |
| Shorter | Regenerates to be more concise |

**Tense:**

| Button | When to use |
| --- | --- |
| Present | Protocol documents, ongoing studies |
| Past | Clinical study reports, completed studies |
| Future | Planned studies, protocol amendments |

### Direct Editing

You can make changes directly in the document — rewrite text, adjust formatting, add or remove content — just as you would in any word processor. Direct edits do not affect your template.

### When to Edit Directly vs. When to Regenerate

| Scenario | Recommended approach |
| --- | --- |
| A few words or sentences need tweaking | Edit directly |
| A paragraph needs a different emphasis or structure | Highlight and regenerate with instructions |
| A section used the wrong source or is missing key content | Regenerate the section |
| You need to add content that doesn't exist in any source | Edit directly |
| A table has incorrect data | Regenerate, or edit the table directly if the fix is small |

## Inserting Content

You can insert tables and reports directly into the document.

1. Click your cursor into the document where you want to insert content
2. Navigate to the **"Add Content"** button in the sidebar
3. Select the table or non-clinical report you want to add
4. Click **"Insert Table"** or **"Insert Report"**

<Note>
  Source files are automatically classified as tables or nonclinical reports during ingestion. Only files classified into one of these types appear in the Add Content list.
</Note>

## Collaborative Editing

Artos supports real-time collaborative editing.

| Mode | What it does |
| --- | --- |
| Fast mode | Changes are synced and visible to all collaborators instantly as you type |
| Strict mode | Your changes are saved locally and only shared when you explicitly save. Competing edits prompt a review when you save. |

You can also turn on **tracked changes** to record every edit with the author's name and timestamp, and leave **inline comments** to flag things for specific collaborators.

## The Editing Mindset

The system is designed to give you more content than you may need. The preferred workflow is to let Artos be comprehensive, then trim.

When reviewing your draft, focus on three questions:

1. **Is the content correct?** Does the information match your sources?
2. **Is the content complete?** Is anything missing? Did Artos pull from the right sources?
3. **Is the content presented the way you want?** Is the structure, emphasis, and level of detail appropriate?

If the answer to #1 or #2 is no, regenerating is usually the fastest path. If the answer to #3 is no, highlight-and-regenerate or direct editing are your best options.

## Template-Level vs. Document-Level Changes

| Situation | Action |
| --- | --- |
| This compound has unique concerns | Post-draft editing |
| Reviewer asked for changes to this document | Post-draft editing |
| Source data for this study is unusual | Post-draft editing |
| All documents should follow this approach | Template change |
| Regulatory requirement affects all documents | Template change |
| Organization-wide style change | Template change |

## Replacing the Document with a Local Copy

You can replace the current document with a locally edited copy (e.g., edited in Word) directly from the document viewer.

<Warning>
  There is currently no way to undo a replacement from the UI. Confirm you have the right document open before uploading. If a replacement needs to be reverted, contact support.
</Warning>
