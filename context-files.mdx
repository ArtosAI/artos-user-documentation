## What are Context Files?

Context files are supplementary inputs you upload alongside source files when generating a document. They guide how specific Artos actions consume source data — which source files to pull from, and where the resulting content should land in the generated document.

There are two context files:

- **List of Tables (LoT)** — used with clinical documents that need tables inserted from TFL sources (CSRs, 2.5s, 2.7s). Pairs with the Table and Summary Placeholder action.
- **Content Plan** — used with documents that need nonclinical study reports summarized into specific sections. Pairs with the Summarize from Nonclinical Report action.

<Warning>
  Keep all required columns in place. Removing any required column from either context file will break matching.
</Warning>

## List of Tables (LoT)

The LoT is used exclusively for clinical documents: CSRs, 2.5s, and 2.7s. Attach one when generating a clinical document that needs at least one table inserted from a TFL source.

<Tip>
  RTF is the preferred format for TFLs — the table is inserted exactly as it appears in the source. PDF and DOCX go through the LLM, so the output may not match the source exactly.
</Tip>

<Warning>
  One table per file. Each TFL must be its own individual file. A single file containing multiple tables won't work.
</Warning>

### Building an LoT

The LoT is an Excel file where each row represents one table. Required columns in order:

| Column | Required | What it holds |
| --- | --- | --- |
| Table Number | Yes | Unique row identifier |
| Table Title | Yes | Source table number, optionally followed by the table name |
| Location | Yes | `In Text` |
| Section | Yes | Section number followed by section name |
| Notes | No | Additional per-row guidance |

Artos matches tables by the **table number** and places them by the **section name** — not the section number. This ensures correct placement even when numbering shifts in the generated document.

### Sample LoT

| Table Number | Table Title | Location | Section | Notes |
| --- | --- | --- | --- | --- |
| Table 1 | 15.8.1.3 | In Text | 8.1 Disposition of Study Participants | Include all data |
| Table 2 | 15.8.3.1 Demographic Baseline Characteristics | In Text | 8.4 Demographic and Other Baseline Characteristics | Include all data |
| Table 3 | 15.11.2.1.1 Adverse Events | In Text | 11.3 Adverse Events | Include all data |

## Content Plan

The Content Plan pairs with the **Summarize from Nonclinical Report** action. Used for IBs and Module 2 nonclinical documents. Attach one when at least one section needs to be populated from a nonclinical study report.

<Warning>
  One study per file. Each nonclinical study report must be its own individual source file. Concatenated files won't work.
</Warning>

### Building a Content Plan

The Content Plan is an Excel file where each row maps one study to one section. Required columns in order:

| Column | Required | What it holds |
| --- | --- | --- |
| Study Name | Yes | Name of the study, as it appears in the source file |
| Study Number | Yes | Unique study identifier |
| Study Type | Yes | Type of study (e.g., Single-dose toxicity, Genotoxicity) |
| Section | Yes | Section number where the summary should be inserted |

<Tip>
  Artos uses all three of Study Name, Study Number, and Study Type when matching against source files. Inconsistencies with what appears in the source report can prevent matching.
</Tip>

### Sample Content Plan

| Study Name | Study Number | Study Type | Section |
| --- | --- | --- | --- |
| 4-Week Oral Toxicity Study in Rats | Study-001 | Repeat-dose toxicity | 5.3 |
| Single-Dose Toxicity Study in Mice | Study-003 | Single-dose toxicity | 5.2 |
| In Vitro Bacterial Reverse Mutation Assay | Study-004 | Genotoxicity | 5.4 |
