## Overview

The Artos Word Add-In brings AI-assisted document editing directly into Microsoft Word — generate content, edit selected text, insert tables and summaries, and chat with an AI assistant with full context of your document, all without leaving Word.

Best used for **post-draft tasks**: refining content, updating tables, and making targeted edits.

## Getting Started

### Installing the Add-In

Available through Microsoft AppSource or via a direct link from your organization. Once installed, find it in the Word ribbon under **Home** or **Insert**.

### Logging In

**Standard:** Open the add-in panel, generate an API key from your Artos account, paste it in, and confirm.

**SSO:** Click **Sign In with SSO** and authenticate through your organisation's sign-in page.

To switch accounts, click **Log Out** in the add-in panel.

### Loading a Document

| Mode | What it does |
| --- | --- |
| Platform Documents | Browse and open a document generated on the Artos platform |
| Use Current Document | Load the Word document you currently have open |

<Note>
  A platform document can only be opened on top of a new Word document. If a document is already open in Word, you will be prompted to open a new one first.
</Note>

Platform Documents can only be edited if they are not already open elsewhere. If a document is locked, the add-in shows you which user currently has it open.

### Choosing a Conversation Mode

| Mode | Best for |
| --- | --- |
| Chat mode (default) | Asking questions, requesting summaries, exploring content |
| Edit mode | Making changes to your document |

## Syncing Changes Back to Artos

When working on a Platform Document, use the **Save** button in the add-in panel to sync edits back to Artos.

<Warning>
  Word's native save (Ctrl\+S) does not sync changes back to Artos. Always use the add-in's Save button.
</Warning>

## What You Can Do

### Generate a List of Abbreviations

1. Switch to **Edit mode** with a section heading already in the document for the LOA
2. Use `/generate-loa` (or type your instruction in plain language)
3. Click **Review suggestion** then **Accept** or **Discard**

The add-in scans your document and cross-references abbreviations against your organisation's master LOA automatically.

### Update an Existing List of Abbreviations

1. Make your new edits to the document in **Edit mode**
2. Use `/update-loa`
3. Review and accept or discard

Only new or changed entries are modified — the entire table is not regenerated.

### Edit Selected Content

1. Switch to **Edit mode** and select the text or table you want to edit
2. Type your instruction (e.g., "Make this more concise", "Remove the Categories column")
3. Click **Review Suggestion** then **Accept** or **Discard**

<Tip>
  For tables, click the table move handle (the small icon at the top-left corner of the table when you hover) to select the entire table cleanly.
</Tip>

### Chat and Summarize

In **Chat mode**:

- "Summarize the findings in Section 5."
- "What adverse events are mentioned in this document?"
- "Does this section contain any inconsistencies with the data in the tables?"

### Insert Tables and Table Summaries

**Supported file formats:** HTML, RTF, PDF, JPEG, PNG

In **Edit mode**, place cursor, type your instruction, attach the file, then Review and Accept or Discard.

What you can ask:

- "Read the table from the attached file and insert it into the document."
- "Summarise the attached table and insert the summary."
- "Filter the attached table for adverse events reported in more than 5% of the study group and insert it."

### Update an Existing Table

In **Edit mode**, select the table, use `/update-table`, attach the file with updated data, then Review and Accept or Discard.

<Tip>
  Marking cells to update with **XX** leads to more reliable outputs — the add-in gets an unambiguous signal of which cells to touch.
</Tip>

## Slash Commands

| Command | What it does |
| --- | --- |
| `/generate-loa` | Generates a List of Abbreviations table |
| `/update-loa` | Updates the List of Abbreviations table |
| `/update-table` | Updates a table using an attached source file |

## Known Limitations

- Chat history is not saved for documents loaded via **Use Current Document**
- Semantic search operations are slower than other operations
- Word's native save does not sync changes back to Artos

## Troubleshooting

| Problem | Solution |
| --- | --- |
| Add-in not responding | Log out and back in. If it persists, close and relaunch Word. |
| Feature not appearing | Log out and back in. If it persists, close and relaunch Word. |
| Responses inconsistent | Make sure your instruction matches one of the supported workflows — general open-ended chat may produce inconsistent results. |
