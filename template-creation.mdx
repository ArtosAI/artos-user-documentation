## What is an Artos Template?

Templates are the foundation of the Artos system. A template is a comprehensive instruction set that teaches Artos how to understand and process your documents.

While you may be familiar with the word "template" from other contexts, in Artos it means something specific: **a set of configured actions and instructions for each section of a document that Artos's AI system can follow to draft documents the way you want.**

Think of template creation as a one-time setup that creates a reusable instruction set. Once created, this template can generate countless documents following your exact specifications.

## Word Template vs. Artos Template

These two terms get used throughout this guide — they sound similar but play very different roles.

<CardGroup cols={2}>
  <Card title="Word Template" icon="file-word">
    The `.docx` file your team already uses to draft documents — the standard template you'd open to start a new CSR, IB, or protocol. It defines what the final document looks like: section headings, layout, tables, boilerplate text, and any drafting instructions written into the file. You bring it with you when setting up Artos.
  </Card>

  <Card title="Artos Template" icon="sliders">
    Something Artos creates on top of your Word template. It's a configuration layer that lives inside the Artos platform, generated automatically from your Word template plus your example files. It defines _how_ Artos drafts each section — which action to run, what content to pull from sources, and any instructions to follow. You review and refine it.
  </Card>
</CardGroup>

## What You'll Need

Before you begin, gather the following:

| Input | What it is | Requirements |
| :-- | :-- | :-- |
| **MS Word Template** | Your document with section headings and instructions | Must be .docx format (not PDF) |
| **Example Documents** | Completed documents that show how content should look | Finished IBs, protocols, CSRs, or other regulatory documents of the same type as the template that you are creating |

## Before You Start: Prepare Your Word Template

Your Word template needs to be properly formatted for Artos to read it correctly. Complete these checks before uploading.

<Steps>
  <Step title="Verify File Format">
    **Your file must be MS Word format (.docx).** You cannot use a PDF.

    Why? Artos extracts complex styling, structure, and formatting information from the Word document that gets lost when you convert to PDF.
  </Step>
  <Step title="Check Section Headings">
    All section headings must use Word styles with the word **"heading"** in the style name.

    | ✓ Works | ✗ Doesn't work |
    | :-- | :-- |
    | Heading 1 | Protocol Title |
    | Heading 2 | Section Name |
    | Section Heading | Body Bold |
    | Protocol Heading 2 | Chapter |

    **How to check:** Click on a heading in your Word document and look at the Styles pane to see what style is applied.
  </Step>
  <Step title="Check Table Styles">
    Tables should use styles with **"table"** or **"caption"** in the style name.

    **Supported styles:**

    - Table Normal
    - Table Grid
    - Caption
    - Custom styles containing "table" or "caption"
  </Step>
  <Step title="Remove Example Content">
    **Do not include example text in your Word template.** Example content belongs in your example files only.

    Exception: You should include template text (boilerplate with placeholders) and template tables (table structures you want Artos to fill in). Template tables should be empty or contain XX in places where data needs to be added.
  </Step>
</Steps>

<Tip>
  **Pre-Upload Checklist**

  1. File is .docx format
  2. All headings use styles with "heading" in the name
  3. Tables use styles with "table" or "caption" in the name
  4. No example content included
</Tip>

## Step-by-Step: Creating Your Template

<Steps>
  <Step title="Navigate to Template Creation">
    1. Log in to the Artos platform
    2. Go to your workspace
    3. Click **"Create Template"**

    <Frame caption="Create Template Button">
      ![Template List](/images/Template-List.png)
    </Frame>
  </Step>
  <Step title="Name Your Template">
    1. Enter a name for your template
    2. Choose a name that clearly identifies the document type (e.g., "Investigator's Brochure Template" or "CSR Template: Phase 2")

    <Frame caption="Create Template - Step 1">
      ![Screenshot 2026 06 30 At 4 35 32 PM](/images/Screenshot-2026-06-30-at-4.35.32-PM.png)
    </Frame>

    <Tip>
      Use a consistent naming convention across your organization so templates are easy to find.
    </Tip>
  </Step>
  <Step title="Select Document Type">
    1. Select the type of document this template will generate
    2. Common options include:
       - Investigator's Brochure (IB)
       - Clinical Study Report (CSR)
       - Nonclinical Overview
       - Protocol
       - Other regulatory documents

    <Tip>
      Template types vary by organization based on the document types agreed to.
    </Tip>

    <Frame caption="Create Template - Step 1">
      ![Screenshot 2026 06 30 At 4 36 34 PM](/images/Screenshot-2026-06-30-at-4.36.34-PM.png)
    </Frame>

    This helps Artos apply the right baseline understanding for your document type.
  </Step>
  <Step title="Upload Your Word Template">
    1. Click file upload field or\*\* \*\*drag and drop your file
    2. Select your prepared .docx file
    3. Wait for the upload to complete

    <Frame caption="Create Template - Step 2">
      ![Screenshot 2026 06 30 At 4 39 23 PM](/images/Screenshot-2026-06-30-at-4.39.23-PM.png)
    </Frame>

    <Frame caption="Create Template - File Uploader">
      ![Frame 52](/images/Frame-52.png)
    </Frame>

    **What happens:** Artos reads your Word template and extracts:

    - All section headings (based on heading styles)
    - Table structures
    - Instructions and template text (based on tags)
    - Formatting and style information
  </Step>
  <Step title="Upload Example Documents">
    1. Click file upload field or drag and drop files
    2. Select a minimum of one and a maximum of five completed documents
    3. Wait for the uploads to complete

    <Frame caption="Create Template - Step 3">
      ![Screenshot 2026 06 30 At 4 42 45 PM](/images/Screenshot-2026-06-30-at-4.42.45-PM.png)
    </Frame>

    **What to upload:**

    | Document Type | Good Example Documents |
    | :-- | :-- |
    | Investigator's Brochure | Completed IBs from previous compounds |
    | Clinical Study Report | Finished CSRs from similar studies |
    | Protocol | Approved protocols from similar studies |

    **Tips for selecting example documents:**

    - Choose documents that represent your organization's best work
    - Use documents that follow your current style guidelines
    - More examples (5 max) help Artos learn better patterns
    - It's better if examples follow the template closely in terms of content.
    - For example, if CSRs for different phases look very different, then maybe create multiple CSR templates with different example files
  </Step>
  <Step title="Generate the Template">
    1. Review your uploads:
       - Word template: ✓ Uploaded
       - Example documents: ✓ Uploaded
    2. Click **"Confirm"**

    **What happens:** Artos processes your inputs and creates an Artos template:

    - Reads headings from your Word template to identify sections
    - Analyzes instructions you've included
    - Extracts content patterns from your example files
    - Maps the structure to create a trained template
    - Auto-assigns actions to each section based on what it learned
  </Step>
  <Step title="Wait for Processing">
    Template generation may take several minutes depending on:

    - Size of your Word template
    - Number and size of example documents
    - Complexity of document structure

    You'll see a progress indicator while Artos processes your inputs.

    <Tip>
      You'll receive a notification when your template is ready.
    </Tip>
  </Step>
  <Step title="Review Your Generated Template">
    Once generation is complete, you'll see the Artos template editor with:

    - **Left panel:** List of all sections (pulled from your Word template headings)
    - **Main panel:** Reference view of your original Word template
    - **Right panel:** Configuration for each section (actions, parameters, instructions)

    **What Artos has done automatically:**

    - Created a section for each heading in your Word template
    - Assigned actions to each section based on the content type
    - Generated initial instructions based on your template text and examples
    - Set default parameters
  </Step>
  <Step title="Review Auto-Generated Configuration">
    Click through each section to review what Artos has configured:

    **For each section, check:**

    | Component | What to review |
    | :-- | :-- |
    | **Actions** | Are the right actions assigned? (Summarize, Copy/Paste, Template Table, etc.) |
    | **Action Instructions** | Are the instructions specific enough? |
    | **Content Toggle** | Should this section generate content or be left blank? Each level sub-heading is treated as its own section. |
    | **Repeat Sections** | Does this section need to create multiple subsections? Based on actual source data e.g., summarizing each non-clinical report as its own sub-section or each primary efficacy endpoint as its own level 4 or 5 sub-heading |

    <Warning>
      **Watch out for duplicate table headings**

      When a table heading and the table itself are both picked up during template generation, Artos may create a 'Use Template Text' action for the heading and a 'Use Template Table' action for the table — but since the 'Use Template Table' action already includes the heading, this results in the heading appearing twice in the generated document. If you notice this, delete the 'Use Template Text' action for the table heading and keep only the Use Template Table action.
    </Warning>
  </Step>
</Steps>

## Troubleshooting Common Issues

| Issue | Cause | Solution |
| :-- | :-- | :-- |
| Sections not appearing | Heading styles don't contain "heading" | Update Word template styles and re-upload |
| Wrong sections detected | Non-heading text using heading styles | Clean up Word template styles |
| Tables not recognized | Table styles don't contain "table" or "caption" | Update Word template table styles |
| Poor action assignments | Example documents don't match template structure | Use example documents that closely match your template |

## What Makes the Auto-Generated Template Better?

Two inputs drive the quality of your Artos template:

1. **Your Word template** provides the structural foundation. The more detail you include, clear headings, specific instructions, boilerplate text, the better Artos understands your document's structure and intent. Artos reads this and converts it into an Artos template, using the context from your instructions and formatting to make intelligent decisions about how each section should be handled.
2. **Your example files** have strong influence on output quality. These are completed documents, finished IBs, CSRs, protocols, or other regulatory documents, that show Artos not just what the document should contain, but how it should be written. Artos learns content patterns, terminology, style conventions, and structural choices from these examples. The better your examples represent how you want your documents to look, the less configuration you'll need.
