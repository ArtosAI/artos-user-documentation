## Overview

Once your Artos template is configured, you use it to generate documents. This is the payoff for your setup work - what might take days to draft manually can be generated in hours.

<Frame>
  ![Image](/images/image-33.png)
</Frame>

## Document Generation Workflow

<Steps>
  <Step title="Name Document & Select Template">
    1. Create a new document in your workspace
    2. Give it a descriptive name

    Select the Artos template you want to use

    <Frame>
      ![Image](/images/image-34.png)
    </Frame>

    <Frame>
      ![Image](/images/image-35.png)
    </Frame>
  </Step>
  <Step title="Upload Source Files">
    Source files are the raw data and documents Artos will draw from to populate your document. These might include protocols, clinical study reports, nonclinical data, TFLs, statistical analysis plans, or other relevant materials.

    <Frame>
      ![Image](/images/image-36.png)
    </Frame>

    **Supported file formats:** .docx, .pdf, .rtf, .xlsx, .csv

    **Examples of source files:**

    | **Document Type** | **Typical Source Files** |
    | --- | --- |
    | **Investigator's Brochure** | Nonclinical study reports, previous versions of the IB |
    | **Clinical Study Report** | TFLs (Tables, Figures, Listings), SAPs, Protocols and amendments |
    | **Nonclinical Overview** | Individual nonclinical study reports, toxicology data |

    <Frame>
      ![Frame 53](/images/Frame-53.png)
    </Frame>

    <Tip>
      **Make your files self-explanatory.** Artos automatically classifies documents according to their type when it processes them. The key principle: **make it clear what each file is and what it contains within the file itself.**

      This might be obvious for standard documents like protocols or clinical study reports, but becomes critical for data files and less structured content.

      **The Junior Person Test:** Before uploading a file, ask yourself: "If I handed this file to a junior person on my team with no other context, would they understand what it is and what it contains?" If the answer is no, consider adding clearer labeling or documentation within the file.
    </Tip>
  </Step>
  <Step title="Add Context Files (Optional)">
    Context files are a special type of input that provide **document-specific information** about what content should go where. Think of context files as instructions that supplement or override the template's default behavior for this particular document.

    **When to use context files:**

    - The document has unique requirements that differ from the template's typical approach
    - You want to specify particular source sections to use for particular document sections
    - You need to provide additional clarification on how to interpret ambiguous source data

    **Context file types by document:**

    | **Document Type** | **Context File** |
    | --- | --- |
    | **Clinical Study Report** | List of Tables |
    | **Investigator's Brochure** | Content Plan |
    | **Nonclinical Overview** | Content Plan |
    | **Other Nonclinical Summaries** | Content Plan |

    **Key distinction:** Context files apply only to the current document generation. If you find yourself using the same context repeatedly, consider adding those instructions directly to your template instead.
  </Step>
  <Step title="Add Document-Level Instructions (Optional)">
    You can add instructions that apply to the entire document or to specific sections. These are useful when study-specific information isn't readily available in the source data, or when you need to guide how Artos interprets the sources for this particular document.

    <Frame>
      ![Image](/images/image-37.png)
    </Frame>

    <CardGroup cols={2}>
      <Card title="Document-wide instructions" icon="file-lines">
        - "Write text at an 8th grade reading level"
        - "Focus only on the 100mg dose formulation throughout"
      </Card>

      <Card title="Section-level instructions" icon="list">
        - "Focus only on specific subgroups for this table"
        - "Emphasize pediatric data in this section"
      </Card>
    </CardGroup>
  </Step>
  <Step title="Kick Off the Generation Job">
    <Frame>
      ![Image](/images/image-38.png)
    </Frame>

    1. Review your selections (template, source files, context files)
    2. Click to start the generation job
    3. Artos begins processing your sources

    <Frame>
      ![Image](/images/image-39.png)
    </Frame>

    <Info>
      **What Happens During Generation**

      When you start a generation job, Artos reads through all your source files and applies the template's actions and instructions to each section. It extracts relevant information, organizes it according to your template structure, and generates formatted content - including tables, summaries, and other structured elements you've specified. After applying any data rules and custom instructions, Artos produces a complete draft document ready for your review.
    </Info>

    **Generation Time**

    Generation time varies based on:

    - Document length and complexity
    - Number and size of source files
    - Template complexity (number of actions and custom instructions)
    - Content composition (text processes faster than tables or figures)

    Plain text processes fastest, followed by tables, then figures and images.
  </Step>
  <Step title="Review Your Draft">
    Once generation is complete, you'll receive an email notification. Your draft document is ready for review in your workspace.
  </Step>
