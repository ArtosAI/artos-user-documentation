## What It Does

Creates a table based on data from other sections within the same document being drafted. This is the tabular counterpart to Create Intra-Document Summary, instead of producing a narrative, it organizes content from elsewhere in the document into a structured table.

## How It Works

Like Create Intra-Document Summary, this action reads from the document Artos is currently building, not from your uploaded source files. It extracts relevant data points from other generated sections and organizes them into a table format.

**Source is the document itself.** The action pulls data from sections that have already been generated within the same document.

**Depends on other sections being generated first.** Artos handles the sequencing automatically, ensuring the sections this action references are generated before the table is built.

**Output is a table.** The action produces structured tabular content. If you need a narrative summary of document content instead, use Create Intra-Document Summary.

**Table structure:** Unlike Use Template Table (where the table structure comes from your Word template), this action can create a table structure based on your instructions. You can specify what columns to include, how to organize the rows, and what data to present.

## Example Instructions

| Weak instruction | Strong instruction |
| --- | --- |
| "Create a summary table" | "Create a table summarizing key efficacy results across all endpoints discussed in the Efficacy Results section. Include columns for: endpoint, analysis population, treatment group, result, p-value, and clinical significance." |
| "Make a table of findings" | "Create a table of nonclinical findings from the sections above. Include columns for: study type, species, dose levels, duration, key findings, and NOAEL. Order by study type." |

**What makes the strong instructions better:** They specify which sections to draw from, what columns the table should include, and how to organize the data. Since the action is creating a table structure (not filling a pre-existing one), your instruction needs to define the layout.

## Configuration Pattern: Summary Table in Discussion Section

A Discussion section might include a summary table that pulls together key results from earlier sections of the document.

| Component | Configuration |
| --- | --- |
| **Action 1** | Create Intra-Document Summary |
| **Action 1 Instructions** | "Synthesize the efficacy and safety findings into a discussion of the overall benefit-risk profile." |
| **Action 2** | Create Intra-Document Table |
| **Action 2 Instructions** | "Create a summary table of primary and secondary endpoint results. Include columns for: endpoint, result, 95% CI, p-value, and clinical interpretation. Pull data from the Efficacy Results section." |
| **Custom Instructions** | "Present the narrative discussion first, followed by the summary table." |

## When to Use This Action

**Use Create Intra-Document Table when:**

- You need a table that summarizes or reorganizes data from other sections in the document
- The table content should reflect what's already been generated, not raw source data
- You need to create a new table structure (not fill an existing one from the Word template)

**Use a different action when:**

- You need to fill a table whose structure is defined in the Word template → use **Use Template Table**
- You need a narrative summary of document content → use **Create Intra-Document Summary**
- You need a table from TFL data with narrative → use **Summarize TFLs**

## Tips

- Be explicit about which columns you want. The action creates the table structure from your instructions, so vague instructions lead to unpredictable table layouts.
- Specify which sections of the document to pull from. "From the Efficacy Results section" is better than "from the document" because it narrows the scope.
- If you need both a narrative and a table from document content in the same section, use both Create Intra-Document Summary and Create Intra-Document Table, and use Custom Instructions to control the order.
