## Roles

### How User Roles Work

Every user in Artos is assigned a **global role**. This role applies consistently across all templates, workspaces, documents, and source files — there are no role variations at the template, workspace, document, or source file level.

When a user is invited to a workspace or template, or when a source file is shared with them, they get access to that resource at their **global permission level**. For example, sharing a template with an Admin gives that user Admin-level rights on the template; sharing the same template with a Member gives them Member-level rights on the template instead. A user's role determines what they can do with every resource they're given access to.

### Permissions by Role

There are 5 user roles - Member, Admin, Author, Owner, and Internal. Internal is a restricted role reserved for Artos staff; its permissions match Owner (see the note below the role tables for assignment rules).

### How sharing works

- When a template or workspace is created, it is only accessible to its creator and users with the role - Owner or Internal. Authors can additionally view any template without it being shared with them. Access for others must be explicitly granted by sharing it with them.
- Source files uploaded by one user are not automatically visible to others, except to users with the role - Owner or Internal. Authors can also view all source files used to create templates without explicit sharing. Other files must be explicitly shared with any user who requires access.
- When you share a template or workspace with someone, you invite them at their global access level. For example - If a Member shares a template with an Admin, the resource would be shared with Admin permissions.

<Tip>
  **Sharing a document doesn't share its source files.** Granting a user access to a generated document does not grant them access to the source files used to create it. If a user needs access to a source file, outside of viewing it from within the generated document, it must be explicitly shared with them.
</Tip>

<Note>
  **In practice:** If your team is collaborating on a template or document, make sure the template, workspace, and any relevant source files have been shared with all users who need them, regardless of their role\*\*.\*\*
</Note>

### Who appears in the share list

Share lists only show users whose access is actually being controlled. Owners and Internal Artos users have access to every template, workspace, document, and source file by default, so sharing with them isn't applicable — they're omitted from share lists. Authors are omitted from template share lists for the same reason — they can already view all templates — but they do appear in workspace and source file share lists. The same applies to share counts.

## Change of Role

Owners can change a user's role at any time. Role management lives in **Settings \> Users & Permissions** — open that page, find the user, and select the desired role from the dropdown next to their name. Owners can move users between **Member**, **Author**, **Admin**, and **Owner** only — the **Internal** role cannot be assigned by Owners through the UI (see the note in the Roles section above). Admins, Authors, and Members cannot change anyone's role.

<Note>
  **Moving to or from the Author role.** When a user is changed to Author (from Owner or Admin), they lose all template create, edit, delete, and rename privileges — including for templates they created or previously edited — and gain view access to all templates and their source files. Their documents, workspaces, and source files are otherwise unaffected. When an Author is changed to Member, they only see the templates that have been explicitly shared with them.
</Note>

<Warning>
  A role change requires a page refresh to take effect on the platform.
</Warning>
