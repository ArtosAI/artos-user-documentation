## What It Does

Populates a table whose structure is defined in your MS Word template. Artos takes the table layout (columns, headers, formatting) from your .docx file and fills it in with data extracted from your source files.

## How It Works

This action operates in two steps. First, it reads the table structure from your Word template, the column headers, row layout, and formatting. Second, it searches your source files for the data needed to fill in each cell, then populates the table accordingly.

**Table structure comes from the Word template.** Artos does not create the table layout, it uses what you've defined in the .docx file. If you want to change the columns, headers, or formatting of a table, you need to edit the Word template and re-upload it.

**Data comes from source files.** The action searches across your uploaded sources to find the information needed for each cell. Your instruction guides what data to look for and how to present it.

**Table accuracy considerations:** Dense, table-based source content (like Schedules of Activities) is harder for the system to extract accurately compared to text-based formats. Where possible, source content that's written out explicitly rather than embedded in complex table structures will produce better results.

## Example Instructions

| Weak instruction | Strong instruction |
| --- | --- |
| "Fill in the table" | "Populate with all Phase 1 studies. Include study number, design, population, dose levels, and key PK parameters. Order by study number." |
| "Complete the demographics table" | "Populate the demographics table with baseline characteristics for the ITT population. Include age, sex, race, weight, and BMI. Present means with standard deviations for continuous variables and counts with percentages for categorical variables." |
| "Add the AE data" | "Populate the adverse events summary table. Include all events with ≥2% incidence in any treatment group. Sort by system organ class, then by frequency within each class." |

**What makes the strong instructions better:** They specify what data to include, how to organize it (sort order), what population to use, and how to present the values. The action knows how to populate a table, your instruction defines what goes in it.

## Configuration Pattern: Safety Section with Narrative and Table

Safety sections commonly pair a Summarize action for the narrative with a Template Table action for the data.

| Component | Configuration |
| --- | --- |
| **Action 1** | Summarize from Source |
| **Action 1 Instructions** | "Summarize safety findings: overall AE profile, SAEs, AEs leading to discontinuation, deaths. Compare treatment groups." |
| **Action 2** | Use Template Table |
| **Action 2 Instructions** | "Populate AE table with events ≥5% in any group. Sort by system organ class, then frequency." |
| **Custom Instructions** | "Present SAEs before common AEs. Include narratives for deaths and drug-related SAEs." |

**Action order matters here.** Artos executes actions top to bottom. If you want the narrative summary to appear before the table, make sure the Summarize action comes first.

## Document-Specific Examples

### Clinical Study Report (CSR)

| Section | Instruction |
| --- | --- |
| Demographics Table | "Populate the demographics and baseline characteristics table. Include age (mean, SD, range), sex, race, ethnicity, weight, height, and BMI. Present for each treatment group and overall population." |
| Adverse Events Summary Table | "Populate the adverse events summary table. Include all events occurring in ≥5% of patients in any treatment group. Sort by system organ class, then by descending frequency. Include incidence counts and percentages for each treatment arm." |
| Prior & Concomitant Medications | "Populate the concomitant medications table. Include medication name, ATC class, and number of patients in each treatment group. Sort by ATC class, then alphabetically by medication name." |
| Subject Disposition | "Populate the subject disposition table. Include number screened, randomized, completed, and discontinued by reason. Present for each treatment arm and overall." |
| PK Parameters Summary | "Populate the pharmacokinetic parameters table. Include Cmax, Tmax, AUC0-t, AUC0-inf, t½, CL/F, and Vd/F. Present geometric means with CV% for each dose group." |

### Investigator's Brochure (IB)

| Section | Instruction |
| --- | --- |
| Nonclinical Studies Summary Table | "Populate with all nonclinical studies conducted. Include study number, study type (single-dose, repeat-dose, genotoxicity, etc.), species, dose levels, duration, key findings, and NOAEL. Order by study type, then chronologically." |
| Clinical Studies Summary Table | "Populate with all completed and ongoing clinical studies. Include study number, phase, design, indication, population, dose range, N enrolled, and status. Order by phase, then by study number." |
| Pharmacokinetic Summary Table | "Populate with PK parameters across clinical studies. Include study number, population, dose, Cmax, AUC, t½, and notable findings. Order by dose level." |

### Nonclinical Overview (NCO)

| Section | Instruction |
| --- | --- |
| Toxicology Studies Table | "Populate with all toxicology studies. Include study type, species, route, dose levels, duration, GLP status, major findings, and NOAEL. Order by study type (single-dose, repeat-dose, genotoxicity, reproductive, carcinogenicity)." |
| Safety Pharmacology Table | "Populate with all safety pharmacology studies. Include study type (CNS, CV, respiratory), species, dose levels, key findings, and NOAEL. Order by organ system." |
| Safety Margins Table | "Populate with safety margins for each species. Include NOAEL, human equivalent dose, proposed clinical dose, and safety margin multiple. Calculate margins based on body surface area." |

## When to Use This Action

**Use Use Template Table when:**

- Your Word template contains a table structure that should be populated with source data
- The table layout (columns, headers, formatting) is fixed and shouldn't change between documents
- You need structured data presentation (demographics, AE summaries, PK parameters, study listings)

**Use a different action when:**

- You need a table created from data within the document itself → use **Create Intra-Document Table**
- You need a table from exported TFL documents with accompanying narrative → use **Summarize TFLs**
- The table content should be copied exactly from a source → use **Copy from Source**

## Tips

- If tables are coming out inaccurate, check whether the source data is in a text-based or table-based format. Text-based source content generally produces better table output than complex nested tables in source documents.
- Be specific about sort order, inclusion thresholds, and which population to use. The most common table issues come from the action making reasonable but incorrect assumptions about these details.
- If you want to consolidate multiple sets of data into single rows rather than splitting them out, specify this explicitly in the instruction.
- To change the table structure itself (add/remove columns, change headers, adjust formatting), edit the Word template and re-upload. The Artos template interface controls the data that fills the table, not the table's structure.
