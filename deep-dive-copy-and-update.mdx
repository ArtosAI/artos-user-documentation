## What It Does

Takes a prior version of the document as its base and updates only what the new source files indicate has changed. Instead of drafting a section from scratch, it starts with existing content and makes targeted revisions based on new data.

## How It Works

This action requires a prior version of the document (or a comparable document from a previous reporting period) as a source file. It reads the existing content, compares it against the new source data, and produces an updated version that reflects only the changes.

**Designed for periodic safety reports and updates.** Copy and Update is built for documents like PBRERs, DSURs, and annual reports, where a prior version exists and the goal is to advance that version with new data rather than rewrite it entirely.

**Preserves unchanged content.** If a section's content hasn't changed based on the new sources, the action keeps it as-is. Only sections with relevant new data get revised.

**Tracks what changed.** The action focuses on what's new — new adverse events, new study data, updated patient numbers — and incorporates that into the existing structure.

## Example Instructions

| Weak instruction | Strong instruction |
| --- | --- |
| "Update this section" | "Update this section based on the new PBRER period data. Revise patient exposure numbers, add any new serious adverse events reported since the last period, and update the benefit-risk conclusion if new data warrants a change." |
| "Add new data" | "Incorporate new clinical study data from the current reporting period. Keep the structure and language from the prior version; update only where the new sources provide different or additional information." |

## Document-Specific Examples

### Periodic Benefit-Risk Evaluation Report (PBRER)

| Section | Instruction |
| --- | --- |
| Patient Exposure | "Update patient exposure figures for the current reporting period. Revise cumulative exposure numbers and add new exposure data by region and formulation if available." |
| Summary of Safety Concerns | "Update the summary of safety concerns to reflect any new signals, label changes, or regulatory actions taken during the reporting period. Keep unchanged sections as-is." |
| Benefit-Risk Evaluation | "Update the benefit-risk narrative to incorporate any new efficacy or safety data from this period. Maintain the structure of the prior version; revise conclusions only if new data supports a change." |

### Development Safety Update Report (DSUR)

| Section | Instruction |
| --- | --- |
| Overview of Development Program | "Update enrollment and exposure numbers for the current reporting period. Add any new studies initiated or completed since the last DSUR." |
| Safety Data | "Update the safety narrative with serious adverse events reported during this reporting period. Revise cumulative exposure numbers and add new findings from ongoing studies." |

## When to Use This Action

**Use Copy and Update when:**

- A prior version of the document exists and should serve as the starting point
- The document type is a periodic report (PBRER, DSUR, annual report)
- The goal is targeted revision, not a full rewrite
- You want to preserve unchanged content from the prior version

**Use a different action when:**

- No prior version exists → use **Summarize from Source** or **Copy from Source**
- The section should be fully rewritten from source data, ignoring the prior version → use **Summarize from Source**
- The content should be copied verbatim from the new source → use **Copy from Source**
